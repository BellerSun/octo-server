# DevBot 任务派发模板

## 派发流程

1. Coda 分析问题，确认修复方案
2. `memory_search` 查找相关知识（已知坑、架构决策、相关代码路径）
3. 组装任务描述（见模板）
4. 发送到 DevBot 频道 `1478587650742026451`

## 任务模板

```
@DevBot [P0/P1/P2] {任务标题}

**问题：** {问题描述}

**修改方案：**
- 文件：{具体文件路径}
- 改动：{具体改什么}

**分支：** `{branch-name}`
**仓库：** `Mininglamp-OSS/{repo}`

**⚠️ 已知约束（来自项目知识库）：**
- {从 memory_search 获取的相关知识}
- {已知的坑和注意事项}

**影响范围：** {哪些功能/模块受影响}

**验收步骤：**
1. {具体验证命令/操作}

**回滚方案：** {如何回滚}
```

## 高风险检查清单

涉及以下内容必须标 `[HIGH-RISK]`：
- [ ] WuKongIM API（token、channel、ban）→ ban 持久化到磁盘，改配置不清除
- [ ] 数据库 migration → 必须 `-- +migrate Up` 开头
- [ ] 权限/认证逻辑
- [ ] 消息路由/转发
- [ ] nullable DB 字段 → 用 `*string` 不用 `string`
